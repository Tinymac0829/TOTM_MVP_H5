## 目标

在真实浏览器环境中，对 `ENG-04` 核心移动手感与 `ENG-05` 最小玩法 UI 的组合链路做一次统一验收，确保 `Story 1` 当前版本具备完整的：

- 开始
- 游玩
- 失败或通关
- 重开或重复游玩

闭环，并作为 `ENG-04` 与 `ENG-05` 的正式收口依据。

## 范围

**包含**：
- `menu -> loading -> playing` 启动链路
- `playing -> paused_fail -> loading -> playing` 失败重开链路
- `playing -> paused_complete -> loading/menu/replay` 通关链路
- `eng04_death_validation` 本地验证关卡中的 Spikes 死亡链路
- HUD 顶部计数、失败弹窗、通关弹窗、加载提示、开始界面
- 弹窗期间输入屏蔽
- `1080x1920` 下 UI 不遮挡核心可玩区域
- 当前仅接入 `story_001` 时的重复游玩 fallback
- 真实浏览器下的点击区域与缩放映射

**不包含**：
- `Story 2`、`Story 3` 真正加载与跳转
- 将 `eng04_death_validation` 视为正式 Story 内容或玩家默认路径
- 新视觉稿、美术润色、动效美化
- `ENG-04` 规则改写
- `Empty(0)` 规则变更
- 调试面板或部署链路验收

## 依赖边界

**已核实**：
- `GameState` 已提供 `playing`、`paused_fail`、`paused_complete`、`onPlayerDead()`、`onStageComplete()`、`getNextStageId()`
- `PlayerController` 需按三层坐标域方案提供四向滑行、`0.1s` 输入缓冲、`5.0 world units/s` 主速度、约 `41.67 tiles/s` 派生显示速度、收集物拾取、死亡回调、出口回调，以及 `stageClearPending`
- `main.js` 已接通 `onCollect -> HUD.addCollectible()`、`onDeath -> GameState.onPlayerDead() + HUD.showFailPopup()`、`onExit -> GameState.onStageComplete() + HUD.showCompletePopup()`
- 当前 `story_001` 不包含 Spikes；失败链路需通过 `?stage=eng04_death_validation` 进入本地验证关卡执行
- 当前 `HUD` 已进入正式实现阶段，但仍允许在不破坏接口契约的前提下继续联调

**推断**：
- 在 `ENG-04` 保持现有事件语义不变时，本清单中的大部分验收项应可直接复用到 `ENG-05` 最终收口

**已确认**：
- `2026-05-01` 真实浏览器回归已覆盖 world-units 运动、短距离过冲、Renderer 坐标适配、`100ms` 输入缓冲、死亡/通关触发时机、输入屏蔽时机和移动收尾时序，结果为 `PASS`

## 联合验收清单

- [ ] 首次进入页面时，默认看到开始界面，而不是直接进入 `story_001`
- [ ] 点击“开始游戏”后，状态按 `menu -> loading -> playing` 进入主玩法
- [ ] 进入 `playing` 后，角色可按 `ENG-04` 规则四向滑行，直到受阻才停
- [ ] `Story 1` 中顶部 HUD 能正确显示 `星 / 币 / 点` 三类计数
- [ ] 经过 Dot、Coin、Star 时，地图中的收集物立即消失，HUD 计数同步更新
- [ ] HUD 顶部栏始终位于画面最上层，不随相机移动错位
- [ ] HUD 顶部栏在 `1080x1920` 下不遮挡核心可玩区
- [ ] 若当前关卡触发死亡，状态按 `playing -> paused_fail` 进入失败弹窗
- [ ] 失败弹窗淡入时长约为 `0.3s`，淡入完成前点击按钮不会触发动作
- [ ] 失败弹窗显示期间，方向输入被屏蔽，角色不会继续移动
- [ ] 点击“重新开始”后，状态按 `paused_fail -> loading -> playing` 返回当前关卡
- [ ] 失败后重开时，角色回到 Enter，收集物恢复，HUD 计数归零
- [ ] 当前关卡通关时，状态按 `playing -> paused_complete` 进入通关弹窗
- [ ] 通关弹窗显示当前关卡的最终 `星 / 币 / 点` 统计
- [ ] 通关弹窗淡入完成前点击按钮不会触发动作
- [ ] 通关弹窗显示期间，方向输入被屏蔽，角色不会继续移动
- [ ] 当前仅接入 `story_001` 时，通关按钮文案和行为与“重复游玩 fallback”口径一致
- [ ] 点击通关按钮后，能重新进入 `story_001`，形成可重复游玩闭环
- [ ] 不同浏览器窗口尺寸下，HUD 按钮点击区域与实际视觉位置一致
- [ ] 若关卡加载失败，界面能回到可恢复状态，并显示清晰的错误提示，不应卡死在不可交互状态

## 手工验证步骤

### 场景 A：开始到游玩

1. 打开页面
2. 观察默认界面是否为开始界面
3. 点击“开始游戏”
4. 观察是否出现加载提示，再进入 `story_001`
5. 进入玩法后，验证角色能正常响应输入

### 场景 B：通关闭环

1. 从开始界面进入 `story_001`
2. 收集部分 Dot、Coin 或 Star
3. 到达 Exit
4. 观察角色是否先完成进入出口格子的动作，再弹出通关层
5. 检查通关弹窗中的最终计数是否正确
6. 点击通关按钮
7. 确认当前版本按 fallback 规则重新进入 `story_001`
8. 确认重进后 HUD 已重置，关卡可再次游玩

### 场景 C：失败闭环

1. 用 `?stage=eng04_death_validation` 进入本地验证关卡后执行死亡验证
2. 观察是否出现失败弹窗，且玩法 fixedUpdate 已暂停
3. 弹窗淡入未完成前尝试点击按钮，确认无响应
4. 淡入完成后点击“重新开始”
5. 确认角色回到 Enter、收集物恢复、HUD 清零
6. 确认从死亡到可重新开始的总体验满足“失败后 `2s` 内可重开”的需求口径

### 场景 D：输入与点击边界

1. 在 `playing` 状态下滑动角色，确认正常移动
2. 在失败或通关弹窗显示期间继续滑动，确认角色不移动
3. 在按钮外区域点击，确认不触发误操作
4. 在不同窗口尺寸下重复点击按钮，确认点击坐标没有明显偏移

## 通过标准

以下条件同时满足时，可认为当前联合验收通过：

- `Story 1` 当前版本具备开始、游玩、通关或失败、重开或重复游玩的完整闭环

- `ENG-05` 的 HUD 状态流、按钮动作、缩放点击区域与当前文档口径一致
- 未发现会阻断继续联调或后续验收的 P0/P1 级问题
